# Workflows

## A — Draft UI copy

1. Name the user's job
2. Draft headline → body → CTA
3. Voice: brave, unifying, authoritative?
4. Tone: concise, simple, to the point, clear, witty if appropriate?
5. Hard rules: person, tense, case, labels, jargon, no em dashes
6. Match modular terms used elsewhere
7. Cut anything that does not help complete the job

## B — Pre-ship checklist

- [ ] User vocabulary, not internal
- [ ] No jargon without plain alternative
- [ ] Consistent terms (segmentation, campaign, audience)
- [ ] Smart default reflected
- [ ] Error / empty / loading states included
- [ ] Case rules applied
- [ ] No em dashes anywhere
- [ ] First person consistent; present tense for behaviour
- [ ] Controls referenced by label text
- [ ] Novice and veteran both understand?
- [ ] Usability scores ≥ 4 if tested

## C — Qualitative validation

Test only what you will change. Prioritize compromises, differentiators, critical user problems.

| Dimension | Questions |
|---|---|
| Utility | Need this? Solves problem? How often used? |
| Usability | Task complete? Time? Mistakes? Repeat without help? |
| Trust | Ease 1-5, trust 1-5 |

Release gate: combined score ≥ 4.

## D — Quantitative validation

- A/B test microcopy
- 5-second / first-glance test
- First-click test
- Default to simpler copy when tied

## E — Cross-channel

1. Same Netcorian voice
2. Same product names (Netcore CEE, Raman, CDP)
3. UI shortest, email longer, docs deepest
4. Never contradict in-app terminology in outbound comms
