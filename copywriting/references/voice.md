# Voice & Principles

Source: Infinity DS zeroheight · Netcorian writing style

## Brand promise

Help brands communicate better with customers through software that is useful, usable, and understandable.

## Products (use correct names)

| Product | Copy angle |
|---|---|
| Netcore CEE | Omnichannel marketing automation at scale |
| Netcore Email API | Speed, reliability, inbox placement |
| Netcore PX | Funnel conversion, guided in-product experiences |
| UNBXD | Relevant search, browse, recommendations |
| Netcore CDP | Unified customer data, consistent cross-channel experience |
| Raman | Sharp, helpful, witty AI insights |

## Voice (stable)

| Marker | Practice |
|---|---|
| Brave | State what the product does without hedging |
| Unifying | Same terms across CEE, CDP, PX, Email API, support, marketing |
| Authoritative | Expert martech colleague, not a chatbot |

## Tone (flexible)

| Trait | Do | Don't |
|---|---|---|
| Concise & precise | "Reach your audience anywhere in the world." | Long vague sentences with multiple implications |
| Simple | "Save changes?" | "Would you like to save your changes?" |
| To the point | "Get insights from Raman" | Over-engineered personalization in UI |
| Clear | Plain language anyone can act on | Jargon plus tooltip to explain jargon |
| Witty (when safe) | Personality in empty states, AI, success | Wit in errors, legal, or destructive confirmations |

## Design principles → copy

**Jobs to be done:** Name the user's task, not the system feature.

**Modular:** Reuse identical labels across Email, Push, SMS (segment, audience, campaign).

**Consistency:** Progressive disclosure. Do not repeat the same help in label, helper, tooltip, and modal.

**Content is king:** User words, not engineer or CxO words.

**Smart defaults:** Fewer choices in copy. "Next step" not "Proceed".

**Natural behaviour:** Familiar patterns. "Send campaign" not "Finalize dispatch".

**Design at scale:** Write for 100%. Gate enterprise-only features in product, not in global copy.
