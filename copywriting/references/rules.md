# Hard Rules

## Audience

- Beginners and industry veterans must both understand the copy.
- UI copy must work without tutorials or videos.
- Technical terms only in API or developer docs.

## Person & tense

- **you/your** for the user.
- Never mix me/my and you/your in the same phrase.
- **Present tense** for product behaviour ("Saves automatically").

## Labels

- Refer to controls by visible label: "Click **Save**" not "Click the primary button".
- One term per concept per flow (do not alternate contacts, users, subscribers).

## Punctuation

- **No em dashes (`—`) anywhere.** Use comma, period, colon, or parentheses.
- Hyphens in compound modifiers are fine.

## Case

| Case | Use for |
|---|---|
| Title Case | Main headings, short introductions |
| Sentence case | Body, helpers, descriptions |
| ALL CAPS | **All button labels in DS 2.0 UI** (`ux-writing.md`) and CEE nudge buttons |

## Transparency

- Say why when something takes time.
- Errors and empty states: what happened + what to do next.

## Ship gate

Utility + usability + usefulness ≥ 4 before customer release when tested.
